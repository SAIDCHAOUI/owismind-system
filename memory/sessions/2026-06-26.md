## Objectif
Aller plus loin que le benchmark standalone (datasets + librairie + scénario dans `OWIsMind_LAB`) et
l'intégrer comme **brique du système**, avec **deux pôles de contrôle** :
1. **Pôle utilisateur (grand public)** : suggérer des questions de benchmark avec la bonne réponse,
   depuis une réponse de chat OU une page dédiée -> construire le golden collaborativement.
2. **Pôle admin (ingénieur)** : un seul endroit pour configurer, lancer et **lire** un benchmark, zéro
   hardcode (tout est réglable, jamais de code à toucher).

## Décision d'archi clé (valide les hésitations du user)
**Les deux pôles ne vivent pas au même endroit -> ça tranche « Dash ? plugin ? autre webapp ? ».**
- Pôle utilisateur = **DANS la webapp Vue (plugin)** : c'est du produit (chat, menu « ... », users).
- Pôle admin = **une webapp DSS STANDARD séparée dans `OWIsMind_LAB`** (HTML/CSS/JS + backend Python) :
  lit les 5 datasets en direct, lit/écrit la variable `benchmark`, lance le scénario. **Zéro Vite, zéro
  zip, zéro body.html, zéro restart** (édition dans le navigateur DSS). PAS Dash (dépend d'un code env,
  charte imparfaite), PAS dashboard natif (ne sait pas éditer la variable), PAS une 2e webapp dans le
  plugin (hérite du build lourd + lecture cross-projet). Le user a explicitement validé « Webapp DSS
  standard dans LAB » (AskUserQuestion). Spec : `docs/superpowers/specs/2026-06-25-benchmark-integration-design.md`.

Découpage en 3 lots indépendants. Le clic « Suggérer pour le benchmark » dans le chat **OUVRE la page
Benchmark préremplie** (comme demandé par le user), pas une modale -> une seule surface de suggestion.
