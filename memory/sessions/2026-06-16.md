## Objectif
Génie logiciel d'abord, modèle ensuite : que l'orchestrateur **utilise nativement** les artefacts
webapp (charts/tables/KPI) au lieu de recopier des tables markdown, qu'il **streame** (narration +
réponse au fil de l'eau), qu'il **escalade** vers Sonnet seulement quand utile (3 modes Éco/Medium/High
choisis par l'user), et améliorer Evidence Studio. Le tout testé local + buildé + zippé pour validation
DSS au matin.

## Décisions clés
- **Streaming = pas de pari sur l'API.** Le SDK Mesh streamé n'expose PAS de preuve qu'il rend les
  `tool_calls` ; je **ne touche pas** au mécanisme `execute()` de décision d'outils (cœur validé).
  Streaming livré par : (1) narration de **préambule** (le texte que le modèle écrit avant un tool-call,
  jusqu'ici jeté, est streamé live) ; (2) **synthèse finale dans un nœud dédié** `execute_streamed()`
  **sans tools** → vrai streaming token par token, fiable (pattern du linéaire d'origine, validé).
- **Nativité par ARCHITECTURE, pas par espoir de prompt** (L060) : le sous-agent renvoyait
  `headline + table markdown` → le petit modèle recopiait la table. Désormais l'orchestrateur hand au
  modèle `headline (table strippée) + DATA compacte JSON + RENDERING HINT` (tool + chart_type + colonnes,
  dérivé de l'intent). Le modèle n'a plus de table à copier → il appelle `show_chart/show_table/show_kpi`.
  **+ filet de sécurité déterministe** : données ≥2 lignes non rendues → **auto-tableau**.
- **Escalade conservatrice 3 modes** (réponse user) : `Éco`=mini seul ; `Medium`=mini + escalade auto de
  la **synthèse** (étape qualité) vers Sonnet sur complexité (heuristique ≥2 signaux) ou échec ;
  `High`=Sonnet partout. Le routing/tool-decisions restent sur mini même en Medium (cheap). Sélecteur
  manuel dans la barre de saisie + plomberie token `⟦owi:mode=…⟧` (prefix → orchestrateur parse+strip).
- **`strict:True` retiré des tool specs** (blocker revue, L061) : OpenAI strict exige toutes les
  propriétés en `required` (optionnelles nullable). Mes specs avaient `title/style/delta` optionnels hors
  `required` → 400 à chaque tour si Mesh forwarde `strict`. Retour exact au pattern validé ; la nativité
  ne dépend pas de `strict` (architecture + descriptions + few-shot suffisent).
