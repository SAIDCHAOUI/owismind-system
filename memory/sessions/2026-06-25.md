## Décisions
- **Architecture Option 1** : projet DSS dédié (`OWIsMind_LAB` assaini) + scénario `Run_Benchmark` + librairie
  partagée ; **appel direct de l'orchestrateur via Mesh** + reconstruction de la réponse complète depuis le
  footer (PAS le chemin webapp HTTP : isolé, prod-safe, vraie latence agent).
- **Stockage = datasets managés** (schémas pensés compatibles SQL `benchmark_*_v1` pour une future section webapp).
- **Scoring = ancre objective déterministe + juge LLM structuré** ; les désaccords ancre↔juge = `needs_review`.
- **Une seule source de config** (variable projet `benchmark`), rien de hardcodé (geste anti-friction user).
- **Agent cross-projet** : `project_key` gère tout (`get_project(KEY).get_llm("agent:xxx")`), pas de préfixe
  dans l'`agent_id`.
