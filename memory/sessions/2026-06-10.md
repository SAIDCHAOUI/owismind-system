# Session 2026-06-10 — Evidence Studio v1 (implémentation complète + audit sécurité + cleaning)

## Objectif
Implémenter **Evidence Studio v1** (3ᵉ colonne de la WebApp) : à la fin d'une génération avec SQL,
ouvrir automatiquement un panneau qui rejoue le SELECT de l'agent **en lecture seule** et montre la
table source avec **les filtres du WHERE appliqués visuellement** (chips), modifiables par l'utilisateur.
But produit = **confiance** : montrer ce que l'agent a réellement fait, pas un bloc SQL illisible.
Volonté user : « step by step », commencer simple, ne PAS recopier la maquette (trop complexe).
Insistance forte : **agents de sécurité + agents critiques** (env Dataiku sensible), éviter tout code
dangereux/lent pour l'instance ; recherches en ligne limitées à la doc Dataiku officielle.

## Fait
Méthode : brainstorming → spec validée → plan (14 tasks) → **subagent-driven** (implémenteur + revue
spec + revue qualité, boucles de correction par batch) → **audit adversarial multi-agents** → cleaning → build/package.
