## Decisions
- Registre/membership/redo dans la VARIABLE (pas de dataset) = zero creation DSS, le launcher est
  l'unique writer (serialise par lock).
- Override = par tentative (cle run_id/question_id/agent_key/mode), donc la consultation/launcher
  envoient le run_id de la ligne, pas un run_id global de benchmark.
- `review_view` liste TOUTES les tentatives ; `detail_view` (consultation) = derniere tentative +
  evolution.
- Lecture plugin par intersection de colonnes = retro-compatibilite des tables non-migrees.
